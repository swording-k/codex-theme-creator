# Codex Theme Creator 上线运营计划

## 上线前检查

- 仓库名称、简介、中文 README 和非官方声明准确。
- 根目录 MIT 许可证与主题素材 CC BY 4.0 声明准确。
- 删除或忽略私人主题、项目名、账号信息和本机路径。
- 四套公开主题全部通过 payload 校验。
- Skill、热安装、恢复、导出和双路由探针全部通过测试。
- 用干净目录解压一个导出 ZIP，按其中 README 完成安装演练。
- GitHub Release 附带版本号、变更说明、已知限制和校验和。

## 首发内容结构

### GitHub

标题：一句话或一张图，给 Codex 生成完整主题

仓库首屏依次展示：

1. 15 秒实机视频：原生界面、输入想法、切换后 New Chat、已有任务。
2. 一句话定位和三个 UI Profile。
3. 一键安装命令与可直接复制的 Codex 提示词。
4. created、installed、active、verified 四种证据说明。
5. 安全、隐私、兼容性和非官方声明。

### 小红书

标题参考：

- 我让 Codex 给自己设计了一整套赛车主题
- 不只是换壁纸：Codex 的侧栏和输入框也能一起换皮肤
- 发一张参考图，让 Codex 自己做主题是什么体验

九宫格顺序：原界面、提示词、背景设计、New Chat、项目选中态、输入框、已有任务、主题包、GitHub 安装方式。

### 抖音 / B 站

30 秒短视频脚本：

1. 0-3 秒：原生界面与成品主题快速对比。
2. 3-8 秒：展示一句中文提示词或拖入参考图。
3. 8-15 秒：展示主题编译与安装状态，不播放冗长终端过程。
4. 15-23 秒：切换 New Chat 和已有任务，重点拍侧栏、按钮、输入框。
5. 23-27 秒：展示 GT、森林、雪湖三套风格。
6. 27-30 秒：GitHub 仓库名和“一键安装，非官方实验”。

长视频重点讲清：为什么普通换背景不够、完整 UI Profile 怎样工作、如何恢复原状、如何分享自己的主题。

### 公众号

文章结构：问题复盘、技术原理、完整创作流程、实机证据、安全边界、开源地址、征集主题。

## 首发文案

```text
我们做了一个 macOS Codex Theme Creator。

你可以只说一句想法，也可以发一张参考图，让 Codex 生成的不只是背景图，
而是一套包含侧栏、项目状态、New Chat、输入框和已有任务页面的完整主题。

项目会把“已生成、已安装、已激活、已验证”分开报告，支持恢复原运行时，
主题包也能导出到 GitHub 分享。

这是非官方开源实验，首版支持 macOS。欢迎提交主题和兼容性反馈。
```

## 数据指标

- GitHub stars、forks、release 下载量。
- Skill 安装成功数和首次创作成功率。
- active 到 verified 的转化率。
- 用户导出主题数和外部 PR 数。
- 安装、可读性、Codex 更新兼容三类问题数量。
- 视频完播率、收藏率和仓库点击率。

## 社区贡献规则

每个主题 PR 必须提供：

- `theme.json`、背景图和主题 README。
- 素材来源与可分发授权说明。
- New Chat 与已有任务的真实截图；截图不得暴露隐私。
- 兼容性探针结果和使用的 Codex 版本。
- 不包含远程资源、任意 JS、账号信息或本机绝对路径。

## 发布边界

- 明确说明项目与 OpenAI 无关。
- 不把 Codex、Porsche、动漫角色或明星名称用作仓库商标或官方暗示。
- 品牌素材可留作私人主题；公开分发前单独审查商标和图片授权。
- AI 生成素材也要检查水印、错误标识和训练平台的商用条款。
- 实机演示不展示私人项目、对话、邮箱、Token 或账号信息。
